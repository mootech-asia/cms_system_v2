# P4 全站截圖驗收報告

> 2026-07-14。設計系統計劃(見 `design-system-plan.md`)最終階段:全站桌機/手機截圖驗收。

## 範圍與方法

- **路由**:全部 22 條(home、6 個遊戲/分類頁、promotion、about、support、12 個會員頁)。
- **狀態**:另加 4 個互動狀態 — slot 廠商遊戲列表(`?provider=`)、promotion 詳情(`?detail=`)、手機漢堡選單(登出態)、會員頁手機版。
- **視口**:桌機 1440×900、手機 390×844,整頁截圖。
- **雙前端**:靜態站(GitHub Pages 版)與 Nuxt 版同輪截圖,共約 100 張。
- **方法**:先以像素差異率(>24/255 視為相異)對每條路由的雙前端截圖評分分流,對高分項逐一目視比對;低分項以縮圖接觸表掃描確認無版面破損。

## 結果

### 通過

- 22 條路由在桌機與手機下雙前端版面一致,無破版、無溢出、無斷字。
- 手機版整頁高度幾乎逐頁對齊(±40px 內),證明區塊結構同步。
- 遊戲分類頁(slot/live/fish/mini-games)雙視口廠商格完全一致;hot-games 直接呈現遊戲列表 + Load More。
- promotion 列表/詳情、slot 廠商徽章視圖、會員全部 12 頁(含 change-password、banking-details)雙前端一致。

### 發現並已修正(本階段)

| 項目 | 問題 | 修正 |
|---|---|---|
| 手機漢堡選單(登出態) | Nuxt 底部為 Login/Register 並排兩顆按鈕,靜態端為「Login 文字列 + 整寬 Register 漸層鈕」 | `AppHeader.vue` 抽屜改為與 `js/mobile.js` 相同的堆疊版式 |

### 殘餘差異(接受,不處理)

| 來源 | 說明 |
|---|---|
| 會員頁內部捲軸 | 靜態版會員頁主內容區為固定視口內捲(`index.html #tweaks`),捲軸佔位造成內容有數 px 水平位移;Nuxt 為整頁捲動。目視等同。 |
| 動畫時序 | 首頁跑馬燈/輪播在截圖瞬間位置不同,推高 home 的像素差異分數。 |
| 次像素累積位移 | 長頁(hot-games 等)卡片高度的次像素差異逐列累積,像素分數偏高但目視無差。 |
| 開發工具浮標 | Nuxt dev 模式截圖含右下角 devtools 浮標,正式建置無此元素。 |
