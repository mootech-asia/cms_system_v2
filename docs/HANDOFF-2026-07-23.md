## 續接方法與指令(給新 session)

**Stage B 逐元件流程(沿用,勿改方法):**
1. 找元件的 utility 叢集 → `grep -nE '^\.CLASS ?\{' factory/win100/assets/css/app.css` 讀出**精確宣告**(勿猜值)。
   注意:`border-*` 色 utility 上**四邊** border-color;`md:`=`@media(min-width:768px)`、`hover:`=`:hover`、
   `sm:`=640、`lg:`=1024;色一律 `rgb(var(--c-*))`。
2. 語意規則寫進 `factory/win100/assets/css/components.css`(body = 精確重組)。
3. markup 換 class:多在 `site.js` 樣板字串裡(`buildGameCard` 等),也有靜態 HTML;用 node 腳本 scope 到該元件的
   區塊做精確字串替換,避免誤中共用同字串的其他 body 內容(header 批踩過此雷)。
4. **改任何 class 前先 `grep site.js`;是 JS hook 就保留(語意 class 並存)。** 現有 hook:`.game-grid`、
   `.vnd-*`、`.fav-toggle`/`.fav-empty`、`.mode-tabs`、`.cms-load-more-*`、`.btn-back`、`.topbar-inner`、`.dd-panel`、
   `aria-label="Switch skin"/"Menu"`、`nav .relative`、`md:hidden`(`nav.md\:hidden button`)、`.is-active`/favorited 狀態 class。
5. 保留所有 `id`/`data-*`/`aria-*`/inline `<svg>`/`href`/文字。清 `router-link-active`/`router-link-exact-active`(死 class,已 0,勿引回)。
